# 투두 앱 기획서

## 개요

이 프로젝트는 단순히 할 일을 저장하는 앱이 아니라, 사용자가 오늘 해야 할 일을 가볍게 정리하고 실행 흐름을 이어갈 수 있게 돕는 리스트형 투두 앱을 만드는 것을 목표로 한다. 전체 UI는 깔끔하고 미니멀하게 유지하되, 자동 상태 분류와 간단한 완료 흐름을 통해 실행 중심의 경험을 제공한다.

## 제품 방향

- 깔끔하고 미니멀한 UI
- 친절하지만 과하지 않은 리스트형 UX
- 빠른 입력과 즉시 실행에 초점
- 사용자가 스스로 우선순위를 정하는 부담을 줄여주는 구조

---

## 1. MVP

### 목표

사용자가 할 일을 빠르게 입력하고, 완료와 이월 흐름까지 한 화면에서 관리할 수 있는 리스트형 투두 앱을 만든다.

### 핵심 경험

1. 할 일을 빠르게 추가한다.
2. 앱이 할 일의 상태를 자동으로 분류한다.
3. 사용자는 할 일 목록을 보고 바로 실행한다.
4. 필요하면 상세 화면에서 메모와 단계를 정리한다.
5. 완료한 일은 아래로 내려가고, 하루 마감 시 성취와 이월 흐름을 확인한다.

### 핵심 기능

- 할 일 빠르게 추가
- 큰 일을 작은 단계로 쪼개기
- 완료 체크
- 내일로 넘기기
- 오늘 성취 한 줄 보기

### 메인 화면 구성

- 오늘 날짜
- 하루마다 바뀌는 리스트형 한 줄 문구
- 할 일 입력 영역
- `할 일 목록` 영역
- `완료한 일` 영역

### 할 일 입력

입력 항목은 아래 3가지로 최소화한다.

- 제목
- 마감일
- 중요도 (`낮음 / 보통 / 높음`)

입력 원칙:

- 저장 즉시 목록에 반영한다.
- 사용자는 빠르게 적을 수 있어야 한다.
- 메모와 세부 단계는 상세 화면에서 다룬다.

### 자동 상태 규칙

앱은 기본적으로 아래 상태를 자동으로 분류한다.

- `오늘 마감`
  - 사용자가 지정한 마감일이 오늘과 같으면 자동 표시
- `늦은 일`
  - 마감일이 지나면 자동 표시
- `여유`
  - 마감일까지 7일 이상 남으면 자동 표시
- `중요도`
  - 사용자가 직접 선택

### 상태 수정

아래 항목은 사용자가 언제든 직접 수정할 수 있다.

- 오늘 마감
- 늦은 일
- 여유
- 중요도

즉, 앱이 먼저 자동 분류해주되 사용자가 직접 고칠 수 있는 구조로 설계한다.

### 할 일 목록 표시 방식

`할 일 목록` 카드에는 아래 정보만 단순하게 표시한다.

- 할 일 제목
- 작은 상태 아이콘

메인 화면은 복잡하지 않게 유지하는 것이 우선이다.

### 완료한 일

- 체크된 할 일은 자동으로 `완료한 일` 영역으로 이동한다.
- 메인 화면 맨 아래에 항상 표시한다.
- 제목에는 취소선 스타일을 적용한다.
- 사용자가 오늘 끝낸 일을 바로 확인할 수 있게 한다.

### 할 일 상세 화면

상세 화면에서는 아래 내용을 다룬다.

- 제목
- 마감일
- 중요도
- 현재 상태
- 메모
- 단계 목록
- `단계 템플릿 추천` 버튼
- `내일로 넘기기` 버튼

상세 화면 역할:

- 메인 화면은 빠르게 보고 실행하는 곳
- 상세 화면은 더 자세히 정리하고 조정하는 곳

### 리스트형 문구

- 리스트형 한 줄 문구는 하루가 지날 때마다 바뀐다.
- 문구 스타일은 아래 2가지 중 선택할 수 있다.
  - `응원형`
  - `담백형`

### 설정 화면(MVP 범위)

설정 화면에서는 아래 항목을 변경할 수 있다.

- 오늘 마감 기준
- 늦은 일 기준
- 여유 기준 일수
- `오늘 마감`, `늦은 일`, `여유` 텍스트 라벨 변경
- 리스트 문구 스타일 선택 (`응원형 / 담백형`)

## 2. MVP 이후

### 목표

기본 투두 앱을 넘어, 사용자가 분류 규칙을 자기 스타일에 맞게 직접 수정하는 개인 맞춤형 생산성 앱으로 확장한다.

### 확장 방향

- 앱이 정한 기본 규칙만 쓰는 것이 아니라 사용자가 규칙을 수정, 삭제, 추가할 수 있도록 확장한다.
- 앱 표현과 라벨까지 사용자 성향에 맞게 바꿀 수 있게 한다.

### 규칙 관리 확장

기본 규칙:

- 오늘 마감
- 늦은 일
- 여유

MVP 이후에는 사용자가 기본 규칙에 대해 아래 작업을 할 수 있다.

- 이름 변경
- 기준 변경
- 삭제

### 사용자 정의 규칙 추가

사용자는 새 규칙을 직접 만들 수 있다.

새 규칙 생성 방식:

- 날짜 기준
- 중요도 기준
- 수동 표시

이 3가지를 조합해서 새 규칙을 만든다.

예시:

- `이번 주 안에 해야 할 일`
- `집중 필요`
- `중요하지만 급하지 않음`

### 규칙별 세부 설정

각 규칙마다 아래 항목을 조정할 수 있다.

- 이름
- 기준
- 아이콘
- 표시 우선순위 반영 여부

즉, 어떤 규칙은 화면 표시만 하고, 어떤 규칙은 목록 정렬이나 강조 기준에도 반영할 수 있다.

### 라벨과 표현 커스터마이징

사용자는 기본 라벨을 원하는 표현으로 바꿀 수 있다.

예시:

- `오늘 마감` -> `오늘 꼭 해야 함`
- `늦은 일` -> `밀린 일`
- `여유` -> `천천히 해도 됨`

즉, 앱 전체 표현을 사용자 스타일에 맞게 바꿀 수 있다.

### 생산성 UX 확장

- 단계 템플릿 종류 확장
- 상태 설명 기능
- 오늘 성취 문구 개인화
- 내일로 넘기기 흐름 세분화
- 더 다양한 상태와 규칙 기반 분류 제공

### 최종 방향

이 앱은 처음에는 할 일을 입력하고, 상태를 보며 완료 흐름을 관리하는 간단한 리스트형 투두 앱으로 시작한다.

이후에는 규칙을 직접 만들고, 표현 방식까지 사용자화하는 개인 맞춤형 생산성 도구로 확장된다.
